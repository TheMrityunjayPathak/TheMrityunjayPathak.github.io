CAR PRICE PREDICTION

GitHub Application

Problem

‚ä≥ In the used car market, buyers and sellers often struggle to determine a fair price for their vehicle.

‚ä≥ This project aims to provide accurate and transparent pricing for used cars by analyzing real-world data.

‚ä≥ It will assist both buyers and sellers make data-driven decisions and ensure fair transactions.


Solution

To address this problem, I built and deployed a complete end-to-end machine learning pipeline :

‚ä≥ Data Collection

‚Üí Scraped a dataset of 2,800+ cars from the web using Selenium and BeautifulSoup.

‚ä≥ Data Optimization

‚Üí Optimized memory consumption of dataset by downcasting data types.

‚Üí Stored the dataset in Parquet format, which compresses data without losing information.

‚Üí It also provides much faster read/write speeds compared to CSV.

‚ä≥ Preprocessing & Modeling

‚Üí Implemented Scikit-learn Pipelines & ColumnTransformer to prevent data leakage.

‚ä≥ API Deployment

‚Üí Deployed the machine learning model as an API using FastAPI, with :

‚Üí /predict endpoint for real-time predictions.

‚Üí /health endpoint for monitoring API status.

‚Üí Input validation & rate limiting for reliability.

‚ä≥ Frontend Integration

‚Üí Designed a HTML/CSS/JS website to send API calls and display predictions in a user-friendly way.

‚ä≥ Containerization

‚Üí Created a multi-stage Dockerfile with .dockerignore for building an optimized and lightweight Docker image.


Impact

‚ä≥ Built and deployed a complete machine learning pipeline as a FastAPI application.

‚ä≥ Reduced dataset memory usage by 90% by downcasting data types and converting to Parquet format.

‚ä≥ Evaluated multiple regression models with cross-validation to identify the best-performing algorithm.

‚ä≥ Achieved 30% lower MAE and 12% higher R2-score compared to the baseline model.

‚ä≥ Improved model stability by 70%, ensuring more stable and reliable predictions.
